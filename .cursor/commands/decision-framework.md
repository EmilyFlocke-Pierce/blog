# Decision Framework Protocol - Sloane Leadership Tools

**Purpose:** Systematic decision-making framework that ensures all choices align with QUILT principles and provide transparent reasoning.

---

## **PROTOCOL STEPS:**

### **1. TIMELINE VIOLATION CHECK (FIRST!)**
- ❌ **FORBIDDEN:** Sprint, Phase, Timeline, ETA, Deadline, Milestone, Weeks, Days, Hours
- ❌ **FORBIDDEN:** Fast, Slow, Quick, Delayed, Rush, Hurry, Time-based language
- ✅ **REQUIRED:** Scope, Dependencies, Deliverables, Blockers, Sequencing
- **If any timeline language detected:** STOP and rewrite in scope-based terms

### **2. LAY OUT ALL OPTIONS**
- Clearly state each available option
- Include brief description of approach
- Ensure all viable alternatives are considered

### **3. ANALYZE AGAINST QUILT PRINCIPLES**
For each option, evaluate:
- **Students First** - How does this serve student needs?
- **Platinum Rule** - How does this treat team members?
- **No Gamification** - Does this avoid unnecessary complexity?
- **Chat-First Interface** - Does this prioritize user experience?
- **Paper Napkin Protocol** - Is this simple and effective?
- **Development Timelines Principle** - Does this use scope-based planning?

### **4. IDENTIFY PRINCIPLE CONFLICTS**
- Which principles are in tension?
- What trade-offs exist between options?
- Which violations are acceptable vs unacceptable?

### **5. MAKE RECOMMENDATION**
- Select option with best principle alignment
- Provide clear rationale based on analysis
- Acknowledge any necessary trade-offs

### **6. PROVIDE CLEAR RATIONALE**
- Explain why this option serves QUILT best
- Address any concerns or objections
- Give team confidence in the decision

---

## **EXAMPLE USAGE:**
```
/decision-framework
"Should we use mock data or real database for frontend development?"
```

**OUTPUT:** Structured analysis with principle evaluation, conflicts identified, and principled recommendation.
