# Analyze Value Rubric Packet

**Context**: We're developing evaluation rubrics for "squishy" abstract/qualitative concepts (strategic judgment, template quality, design coherence, process effectiveness) rather than just concrete technical metrics. Once we can evaluate these higher-level thinking concepts, we can cascade that structure down to more concrete evaluations.

**Task**: Please review `docs/value_rubric_packet.pdf` and:
1. Identify rubric structures/frameworks that could work for evaluating abstract/qualitative concepts
2. Extract key patterns (anchor descriptions, scale levels, criteria organization)
3. Note anything useful for creating rubrics that assess "does this feel right?" type questions
4. Suggest how we might adapt these structures for our context (agent work, templates, strategic decisions)

Focus on rubrics that help evaluate things that are hard to quantify but important to assess consistently.
